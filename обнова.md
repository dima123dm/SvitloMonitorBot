### Формат графіків
- **Слоти по 30 хвилин**: `"00:00"`, `"00:30"`, ..., `"23:30"` (48 слотів)
- **Статуси**: `1` = світло є, `2` = гарантоване відключення, `3` = можливе відключення
- **Дати**: `YYYY-MM-DD` (сьогодні + завтра)

### ✅ Сумісність з поточним кодом
Формат **ідентичний** поточному API → `parse_intervals()`, `calculate_off_hours()`, `format_message()` працюють **без змін**.

### 🆕 Нове поле: `emergency`
- `true` = екстрені відключення (наразі Одеська область)
- Можна додати сповіщення "⚡ Увага! В регіоні діють екстрені відключення"

### ⚠️ Різниця в назві Хмельницької
- **Поточне API**: `"Хмельницька"`
- **Нове API**: `"Хмельницька область"`
- Потрібна нормалізація при порівнянні

## 🏗️ Архітектура Failover-системи

### Логіка перемикання
```
┌─────────────────────┐
│   Нове DTEK API     │ ← ОСНОВНЕ ДЖЕРЕЛО
│   (svitlo-proxy)    │
└─────────┬───────────┘
          │ Працює?
          ├── ДА → використовуємо
          └── НІ → рахуємо час простою
                    │
                    ├── < 2 години → повторюємо спроби
                    └── ≥ 2 години → ПЕРЕМИКАННЯ
                              │
                    ┌─────────▼───────────┐
                    │   Резервне API      │ ← РЕЗЕРВ
                    │   (поточне)         │
                    └─────────┬───────────┘
                              │ Через 24 години
                              │ перевіряємо основне
                              ├── Працює → повертаємось
                              └── Ні → залишаємось на резерві
                                        (повторюємо через 24г)
```

### Стан системи (зберігається в пам'яті)
```python
api_state = {
    "active_source": "primary",    # "primary" або "backup"
    "primary_down_since": None,    # datetime коли впало основне
    "backup_down_since": None,     # datetime коли впало резервне
    "last_primary_check": None,    # коли останній раз перевіряли основне (з резерву)
    "last_backup_check": None,     # коли останній раз перевіряли резервне (з основного)
    "consecutive_failures": 0,     # підряд невдачі поточного джерела
}
```

## 📝 Зміни по файлах

### 1. `config.py` — додати URL нового API
```python
# Основне API (DTEK Proxy)
PRIMARY_API_URL = "https://dtek-api.svitlo-proxy.workers.dev/"

# Резервне API (поточне джерело)
BACKUP_API_URL = "Апі_джерела"   # <- поточне API_URL

# Таймінги failover (в секундах)
FAILOVER_TIMEOUT = 7200       # 2 години до перемикання
RECOVERY_CHECK_INTERVAL = 86400  # 24 години до перевірки основного
```

### 2. `api_utils.py` — основні зміни

#### Нова функція: `fetch_primary_api()`
```python
async def fetch_primary_api():
    """Запит до основного DTEK Proxy API."""
    try:
        async with aiohttp.ClientSession() as session:
            async with session.get(PRIMARY_API_URL, timeout=15) as response:
                if response.status == 200:
                    raw = await response.json()
                    # Подвійний парсинг (body = JSON-рядок)
                    data = json.loads(raw.get('body', '{}'))
                    # Нормалізація назв регіонів
                    return normalize_region_names(data)
    except Exception as e:
        print(f"❌ Primary API Error: {e}")
    return None
```

#### Нова функція: `normalize_region_names(data)`
```python
# Маппінг для сумісності:
REGION_NAME_MAP = {
    "Хмельницька область": "Хмельницька",
    # Додати інші якщо знайдуться відмінності
}

def normalize_region_names(data):
    """Приводить назви регіонів до стандарту бота."""
    if not data or 'regions' not in data:
        return data
    for region in data['regions']:
        original = region.get('name_ua', '')
        region['name_ua'] = REGION_NAME_MAP.get(original, original)
    return data
```

#### Оновлена `fetch_api_data()` — з failover логікою
```python
async def fetch_api_data():
    """ГОЛОВНА ФУНКЦІЯ з failover."""
    global api_state
    now = datetime.now()
    
    if api_state["active_source"] == "primary":
        # Спроба основного API
        data = await fetch_primary_api()
        if data:
            api_state["consecutive_failures"] = 0
            api_state["primary_down_since"] = None
        else:
            # Основне не працює
            if api_state["primary_down_since"] is None:
                api_state["primary_down_since"] = now
            
            elapsed = (now - api_state["primary_down_since"]).total_seconds()
            if elapsed >= FAILOVER_TIMEOUT:
                # Перемикання на резерв
                print("🔄 Перемикання на РЕЗЕРВНЕ API!")
                api_state["active_source"] = "backup"
                api_state["last_primary_check"] = now
                data = await fetch_backup_api()
            else:
                # Ще чекаємо, пробуємо резерв як fallback
                data = await fetch_backup_api()
    
    else:  # active_source == "backup"
        # Перевіряємо чи час повертатись на основне
        if api_state["last_primary_check"]:
            elapsed = (now - api_state["last_primary_check"]).total_seconds()
            if elapsed >= RECOVERY_CHECK_INTERVAL:
                # Пробуємо основне
                primary_data = await fetch_primary_api()
                if primary_data:
                    print("✅ Основне API відновлено! Повертаємось.")
                    api_state["active_source"] = "primary"
                    api_state["primary_down_since"] = None
                    data = primary_data
                else:
                    api_state["last_primary_check"] = now
                    data = await fetch_backup_api()
            else:
                data = await fetch_backup_api()
        else:
            data = await fetch_backup_api()
    
    # Інтеграція HOE сайту (як раніше)
    ...
    
    return data
```
