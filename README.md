**Svitlo Monitor Bot** — це професійне асинхронне рішення для автоматизованого моніторингу графіків відключень електроенергії. Бот забезпечує надійну доставку інформації завдяки системі подвійного резервування API та інтелектуальному планувальнику завдань.

---

## 🌟 Ключові можливості

### 👤 Для користувачів
*   **Інтелектуальні сповіщення**:
    *   `Pre-alert`: Попередження за 5, 15, 30 або 60 хвилин до відключення.
    *   `Post-alert`: Сповіщення про відновлення електропостачання.
    *   `Moment-alert`: Повідомлення точно в момент перемикання за графіком.
    *   `Update-alert`: Повідомлення про раптові зміни в графіках протягом дня.
*   **Ранкове зведення**: Автоматичне надсилання персоналізованого графіку о 06:00.
*   **Візуалізація**: Вибір режимів відображення ("Чорний" — коли темно, "Зелений" — коли світло).
*   **Аналітика**: Статистика відключень за останні 7 днів.

### 📢 Для груп та каналів
*   **Легка інтеграція**: Команда `/addtogroup` з автоматичним генератором посилань.
*   **Адмін-інтерфейс у приватних повідомленнях**: Спеціальна система керування каналами через Deep Linking.
*   **Повна синхронізація**: Налаштування груп ідентичні особистим — таймери, типи сповіщень та режими.

### 👨‍💻 Для адміністраторів
*   **Вбудована адмін-панель**: Керування ботом безпосередньо через Telegram.
*   **Система тікетів**: Вбудована техпідтримка для зв'язку з користувачами.
*   **Автоматичні бекапи**: Щоденне створення та відправка бекапу бази даних о 03:00.

## 🛠 Технічна архітектура

### 🔄 Dual-API Failover System
Проект використовує просунуту систему відмовостійкості:
1.  **Primary API**: Основне джерело даних (DTEK Proxy).
2.  **Backup API**: Резервний вузол, який активується автоматично при виникненні помилок в основному джерелі.
3.  **Автоматичне відновлення**: Система періодично перевіряє доступність основного API для повернення в стандартний режим роботи.

### 🧠 Планувальник (Scheduler)
Ядро бота, що працює в асинхронному фоновому режимі:
*   Виконує превентивну перевірку черг для уникання масового спаму.
*   Оптимізує розсилку для тисяч користувачів одночасно.

## 📂 Структура проекту

| Файл | Опис |
| :--- | :--- |
| `main.py` | Точка входу, ініціалізація сервісів та запуск бота. |
| `handlers.py` | Вся логіка взаємодії, меню, адмін-функції та керування групами. |
| `scheduler.py` | Фонова обробка: оновлення даних, розсилки, бекапи. |
| `api_utils.py` | Модуль парсингу, обробки API та роботи з часовими інтервалами. |
| `database.py` | Асинхронний шар роботи з SQLite (користувачі, черги, статистика). |
| `config.py` | Менеджер конфігурації через змінні оточення. |

## 🚀 Встановлення

### 1. Клонування репозиторію
```bash
git clone https://github.com/mriamys/SvitloMonitorBot.git
cd SvitloMonitorBot
```

### 2. Налаштування середовища
```bash
python -m venv venv
# Windows:
venv\Scripts\activate
# Linux/macOS:
source venv/bin/activate
```

### 3. Встановлення залежностей
```bash
pip install -r requirements.txt
```

## 📝 Конфігурація

Створіть файл `.env` на основі `.env.example`:

| Змінна | Призначення |
| :--- | :--- |
| `BOT_TOKEN` | Токен від @BotFather |
| `ADMIN_IDS` | ID адміністраторів (через кому) |
| `PRIMARY_API_URL` | Основне джерело даних |
| `BACKUP_API_URL` | Резервне джерело даних |
| `UPDATE_INTERVAL` | Частота оновлення даних (сек) |
| `FAILOVER_TIMEOUT` | Час до перемикання на резерв (сек) |
